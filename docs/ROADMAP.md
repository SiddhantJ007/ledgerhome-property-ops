# Roadmap

This roadmap is intentionally realistic. It does not try to make the project sound “complete.” It shows what would actually be worth doing next if LedgerHome kept evolving as a product and engineering system.

## Short-Term Improvements

- improve UI and UX polish across web flows
- add better empty, loading, and error states
- add more seed data so the demo feels richer
- add Playwright E2E tests
- improve mobile responsiveness and browser QA coverage

## Product Improvements

- Stripe payment tracking
- automated rent reminders
- maintenance request status notifications
- tenant document uploads
- stronger lease management workflows
- analytics and operational dashboard reporting

## Engineering Improvements

- Supabase local development setup
- deeper RLS policy tests
- audit logs
- monitoring and error tracking
- branch protection
- preview deployment QA checklist

## Security Improvements

- MFA
- rate limiting
- stricter role management
- admin activity logs
- more explicit security review around public demo exposure

## Delivery Philosophy

I prefer roadmap items that preserve clarity about what is already real versus what is still future work. The point of this roadmap is not to inflate scope. The point is to show that I know where the product and engineering gaps still are and how I would close them next.
